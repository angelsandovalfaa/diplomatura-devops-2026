# Plan de Estudio DevOps (Organizado por Fases)

Este plan usa los 9 repositorios base en un orden pedagógico: primero fundamentos, luego práctica intensiva, después proyectos reales y finalmente preparación laboral.

## Estructura General (12 Semanas)

- Duración: 12 semanas
- Dedicación recomendada: 1.5 a 2 horas por día, 5 días por semana
- Método: 30% teoría + 70% práctica
- Evidencia mínima semanal: 1 resumen + 1 práctica + 1 commit/log

---

## Fase 1: Dirección y Fundamentos (Semanas 1-3)

### Semana 1
- Repos: 9 (DevOps Roadmap) + 4 (Into the DevOps)
- Objetivo: entender el mapa completo de habilidades DevOps.
- Entregable: documento de ruta personal (stack, herramientas, metas).

### Semana 2
- Repos: 1 (Cloud-DevOps Learning Resources)
- Objetivo: consolidar Linux, redes, Git, contenedores y cloud básico.
- Entregable: apuntes estructurados por tema + comandos clave.

### Semana 3
- Repos: 3 (DevOps Exercises)
- Objetivo: resolver ejercicios de Linux/Git/Docker.
- Entregable: 25+ ejercicios documentados.

---

## Fase 2: Automatización y Plataforma (Semanas 4-7)

### Semana 4
- Repos: 3 + 4
- Objetivo: CI/CD básico con GitHub Actions o Jenkins.
- Entregable: pipeline funcional con build y test.

### Semana 5
- Repos: 3 + 1
- Objetivo: Kubernetes básico + observabilidad inicial.
- Entregable: app desplegada en K8s local (minikube/kind).

### Semana 6
- Repos: 3 + 5
- Objetivo: Terraform e infraestructura como código.
- Entregable: módulo IaC inicial (entorno dev simple).

### Semana 7
- Repos: 5
- Objetivo: integración CI/CD + IaC + deploy.
- Entregable: mini proyecto DevOps end-to-end.

---

## Fase 3: Arquitectura, Seguridad y Especialización (Semanas 8-10)

### Semana 8
- Repos: 2 (System Design Primer)
- Objetivo: pensar sistemas escalables y trade-offs.
- Entregable: 2 diseños de arquitectura en markdown.

### Semana 9
- Repos: 6 (Cloud Native Security)
- Objetivo: incorporar prácticas DevSecOps.
- Entregable: checklist de hardening para tu proyecto.

### Semana 10
- Repos: 7 (MLOps Basics)
- Objetivo: conocer pipeline de modelos (entrenar, empaquetar, desplegar, monitorear).
- Entregable: mini flujo MLOps documentado.

---

## Fase 4: Cierre Profesional (Semanas 11-12)

### Semana 11
- Repos: 5 + 8
- Objetivo: mejorar proyecto portfolio y preparar storytelling técnico.
- Entregable: README profesional + diagrama + decisiones técnicas.

### Semana 12
- Repos: 8 + 3
- Objetivo: simulación de entrevistas DevOps.
- Entregable: banco de respuestas + mock interview.

## Semana 1 (Detalle Diario - Empezamos Hoy)

### Día 1 (Hoy)
- Objetivo: definir rumbo y encender la práctica.
- Tareas:
  1. Leer README de Repo 9 (DevOps Roadmap).
  2. Definir stack inicial: `AWS + Docker + GitHub Actions + Kubernetes + Terraform`.
  3. Leer índice principal de Repo 4 (Into the DevOps).
  4. Resolver 3 ejercicios básicos de Repo 3 (Linux/Git).
- Evidencia mínima:
  - 1 nota de decisiones técnicas.
  - 1 bloque de comandos usados.

### Día 2
- Objetivo: base Linux/Git fuerte.
- Tareas:
  1. Resolver 5-8 ejercicios Linux/Git de Repo 3.
  2. Documentar comandos importantes y errores comunes.
- Evidencia mínima:
  - 1 resumen corto + ejercicios resueltos.

### Día 3
- Objetivo: Docker fundamentos.
- Tareas:
  1. Revisar contenido Docker (Repo 1 o 4).
  2. Resolver 5 ejercicios Docker de Repo 3.
  3. Levantar un contenedor simple y validar logs.
- Evidencia mínima:
  - 1 `docker run` funcional + capturas/logs.

### Día 4
- Objetivo: CI/CD introducción.
- Tareas:
  1. Revisar sección CI/CD en Repo 4.
  2. Crear flujo inicial en GitHub Actions (build/test básico).
- Evidencia mínima:
  - 1 workflow y ejecución registrada.

### Día 5
- Objetivo: cierre semanal con orden.
- Tareas:
  1. Consolidar apuntes de la semana.
  2. Identificar 3 fortalezas y 3 brechas.
  3. Definir plan exacto de Semana 2.
- Evidencia mínima:
  - 1 retrospectiva semanal.
